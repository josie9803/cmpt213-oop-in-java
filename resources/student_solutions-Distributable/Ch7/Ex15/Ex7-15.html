In package java.lang, the only public classes that implement equals directly are Object, the wrapper classes and class String and StackTraceElement. All appear to meet the axioms. Some of them use instanceof, but they are final classes so there are no problems with subclasses. 


In contrast, package java.awt has over 80 public classes listed in the API documentation. Of these, four final classes (AlphaComposite, DisplayMode, JobAttributes, PageAttributes) implement equals correctly. The following non-final classes implement equals using instanceof rather than getClass: BasicStroke, Color, Dimension, Insets, MenuShortcut, Point, Rectangle, RenderingHints. This is not a problem for the Java engineers since these classes are not subclassed in a way that would change the semantics of equals. But it is a problem for you if you wanted to, for example, have a subclass LabeledPoint or ColoredPoint of Point. You would not be able to redefine the equals method to take labels or colors into account.

Font.equals is bizarre: it casts and catches a ClassCastException if the cast fails. Don't code like that.
