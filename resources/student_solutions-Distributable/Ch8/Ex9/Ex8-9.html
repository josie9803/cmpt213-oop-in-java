Interfaces would be as follows:
  • ReadOnlyCollection does not have add/remove
  • Collection has add and remove (not optional)

  • ReadOnlyList does not have add/remove, set
  • ModifiableList has set, but not add/remove
  • List has add/remove and set (not optional)

  • ReadOnlySet does not have add/remove
  • Set has add/remove, but not set (like current Set)

  • ReadOnlyIterator does not have remove
  • Iteraltor

  • ReadOnlyListIterator does not have remove and set
  • ModifiableListIterator has set but not add/remove
  • ListIteraltor
Note that "modifiable" makes sense for indexed collections only.

Abstract classes would be as follows:

  • AbstractReadOnlyCollection
  • AbstractCollection extends AbstractReadOnlyCollection

  • AbstractReadOnlyList
  • AbstractModifiableList extends AbstractReadOnlyList
  • AbstractList extends AbstractModifiableList and copies operations from AbstractCollection.

Note that AbstractList could extend AbstractCollection and copy operations from AbstractModifiableList.

In class Arrays, method asList should be modified to return a ModifiableList.

Class Collections should add method modifiableList, and method unmodifiableList should be modified to return a ReadOnlyList.
