• Example 1
    1. Thread 1 executes removeFirst and completes its time slice after the value of r is assigned, but before before head++ is executed. The second thread gains control.
    2. Thread 2 executes removeFirst which assigns to r to the same value that was assigned in Thread 1. Thread 2 completes execution of removeFirst and gives up control.
    3. Thread 1 resumes execution and completes removeFirst which returns the same value that was returned from removeFirst in thread 2. Since head was incremented twice, two values were removed from the queue, however, only one of those values was returned by removeFirst.

  • Example 2
    1. Thread 1 executes removeFirst and completes its time slice before size-- is executed. The second thread gains control.
    2. Thread 2 executes isEmpty which returns false. Thread 2 then gives up control.
    3. Thread 1 resumes execution and executes size-- which sets size to 0 (ie makes the queue empty). Thread 1 gives up control.
    4. Thread 2 regains control and treats the queue as not being empty, even though it is.

  • Example 3
    1. Thread 1 executes add and completes its time slice before size++ is executed. The second thread gains control.
    2. Thread 2 executes isFull which returns false. Thread 2 then gives up control.
    3. Thread 1 resumes execution and executes size++ which sets size to elements.length (ie makes the queue full). Thread 1 gives up control.
    4. Thread 2 regains control and treats the queue as not being full, even though it is.
